## Firebase

- Google에서 만든 서비스로 웹 및 모바일 앱을 제작하기 위한 도구들을 제공
- 데이터베이스로 `Firestore`를 사용하며 키:값 쌍의 컬렉션으로 이루어진  "문서" 단위로 데이터를 저장한다.
- 내부적으로는 GCP(Google Cloud Platform)를 사용한다

https://firebase.google.com/pricing?hl=ko&_gl=1*76voio*_up*MQ..*_ga*MTMwMDg3MDE5OS4xNzIzOTgzNDkx*_ga_CW55HF8NVT*MTcyMzk4MzQ5MC4xLjAuMTcyMzk4MzQ5MC4wLjAuMA..

## Supabase

- 자칭 **Firebase 대안**으로 부르는 오픈소스 기반 서비스.
- 데이터베이스로 RDB의 한 종류인 PostgreSQL을 사용한다.
- 오픈소스 기반으로 종속성이 없다
- 고전적이고 가장 널리 사용되는 SQL을 사용하여 데이터 베이스를 쿼리할 수 있다

https://supabase.com/pricing#compare-plans

## 데이터 베이스

### Firebase

- 1 GiB 용량
- 아웃 바운스 10GiB/month
- 쓰기 작업 20,000/day
- 읽기 작업 50,000/day
- 삭제 작업 20,000/day

### Supabase

- 전용 데이터베이스
- 500 MB 용량
- 대역폭 5GB
- 무제한 API 호출
- 일주일 간 활동이 없으면 일시정지

### Appwrite

- 무제한 문서
- 읽기 및 쓰기 무제한
